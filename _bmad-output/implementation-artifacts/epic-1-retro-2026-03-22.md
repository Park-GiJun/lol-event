## 개선할 것들

1. **shadcn CLI 산출물 자동 신뢰 금지**
   - `npx shadcn@latest add` 실행 시 `frontend/@/components/ui/` (리터럴 `@` 디렉토리)에 파일 생성 → 수동으로 `src/components/ui/`로 이동 필요
   - 생성된 파일에 `import * as React` 누락 (`skeleton.tsx`)
   - displayName 오타 포함 (`"BreadcrumbElipssis"` → `"BreadcrumbEllipsis"`)

2. **shadcn 의존 패키지 자동 설치 안 됨**
   - `tailwindcss-animate` 미설치 → Popover 애니메이션 동작 안 함 (코드 리뷰 Patch로 발견)
   - shadcn 컴포넌트 추가 시 README/changelog에서 의존 패키지 수동 확인 필요

3. **shadcn CSS 변수 시스템 미정의 (Defer)**
   - `--background`, `--foreground`, `--muted`, `--card` 등 shadcn이 의존하는 CSS 변수가 프로젝트에 없음
   - shadcn 컴포넌트들이 투명 배경으로 렌더링될 수 있음
   - 의도적으로 Defer — Epic 2 Story 2.1에서 해결 예정

## 핵심 인사이트

1. **스펙 파일 Dev Notes의 품질이 구현 품질을 결정** — 이전 스토리 학습사항을 Dev Notes에 문서화할수록 Patch 건수 감소
2. **shadcn은 외부 도구 산출물** — CLI 생성 파일은 항상 수동 검증 체크리스트 적용
3. **사전 컨텍스트 분석 필수** — 스토리 구현 전 기존 코드 상태 확인이 중복 작업 방지에 핵심
4. **`verbatimModuleSyntax` + `noUnusedLocals` 조합** — 처음에는 까다롭지만 코드 품질을 자동으로 강제하는 좋은 도구
