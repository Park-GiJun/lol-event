## 아이디어 전체 목록

### 테마 1: 설계 원칙

**[Architecture #4]: Dashboard vs Electron 역할 분리**
_Concept:_ Dashboard=역사적 통계 허브, Electron=현재 진행형 전투 지휘소. 두 앱의 시간축이 완전히 다름.
_Novelty:_ 같은 데이터 소스를 쓰지만 목적이 달라 화면 구성 방향이 전혀 다름.

**[Context #5]: 내전 전용 앱의 본질**
_Concept:_ 유저들은 서로를 이미 알고 있음. UI는 "새로운 정보 탐색"이 아닌 "내가 아는 걸 확인/증명"하는 경험 제공.
_Novelty:_ 일반 통계 앱과 달리 커뮤니티 맥락이 데이터에 감정적 의미를 부여함.

**[Design Rule #11]: 숫자 우선 원칙**
_Concept:_ 승률, Elo, 게임 수 등 모든 지표는 숫자로 표현. 이모지/아이콘으로 대체하지 않음.
_Novelty:_ 데이터 친화적 사용자층에게 꾸밈보다 정확성이 우선.

**[Foundation #16]: 앱 목적 완전 분리**
_Concept:_ Dashboard=통계/리포트 허브, Electron=현재 매치에 집중하는 실시간 도구.
_Novelty:_ 두 앱이 역할이 명확히 달라 각자 최적화된 UX를 가질 수 있음.

**[Foundation #18]: 데이터 유지 + UI/UX 백지 리디자인**
_Concept:_ 기존 데이터 모델과 API는 최대한 활용. 화면 구성/레이아웃/인터랙션만 완전 재설계.
_Novelty:_ 데이터 작업 최소화 + UX 혁신 극대화. 리스크 낮추고 개선 폭 최대화.

**[Design Rule #21]: 시즌 없음, 전체 누적 데이터**
_Concept:_ 모든 기록은 시즌 구분 없이 전체 누적 기준. 명예의 전당도 "역대 최고" 단일 기준.
_Novelty:_ 복잡한 시즌 로직 불필요. 기록은 깨지기 전까지 영원히 유지됨.

**[Design Rule #22]: 관리 페이지 전체 유지**
_Concept:_ MonitoringPage, SyncPage, LcuPage, AdminPage 등 관리용 페이지 모두 유지. 일반 유저 화면과 분리된 관리자 영역.
_Novelty:_ UI 리디자인 범위에서 관리 페이지 제외 — 통계/리포트 화면에만 집중.

### 테마 2: Dashboard UI/UX

**[UX #1]: 챔피언 승률 리더보드 위젯**
_Concept:_ 홈 화면 상단에 이번 주 Top N 챔피언 카드 배치. 픽률/승률/밴률을 한눈에 보여주는 숫자 기반 랭킹.
_Novelty:_ 단순 테이블이 아닌 챔피언 스플래시 아트 + 수치로 직관적 표현.

**[UX #2]: 밴픽 트렌드 표시**
_Concept:_ "많이 밴되는 챔피언" 섹션 — 밴률 변화 추이를 수치로 표시.
_Novelty:_ 단순 순위가 아닌 트렌드 방향으로 메타 변화를 직감적으로 파악.

**[UX #3]: 유저 Elo 리더보드**
_Concept:_ 전체 유저 Elo 랭킹을 홈에서 바로 접근. 자신의 순위를 리더보드 중간에 하이라이트.
_Novelty:_ "나는 어디쯤인가" 맥락 제공으로 동기부여.

**[Adapt #13]: OP.GG 티어표 스타일 적용**
_Concept:_ 챔피언을 S/A/B/C 티어로 분류해서 밴픽 우선순위를 한눈에 파악. 승률+픽률 기반 자동 티어 산정.
_Novelty:_ 유저들이 이미 OP.GG에서 익숙한 패턴 — 학습 비용 0.

**[Adapt #14]: 내전 자체 챔피언 티어표**
_Concept:_ 외부 메타가 아닌 우리 내전 데이터로 만든 자체 티어표. "이 리그에서는 야스오가 S티어" 같은 내전 전용 메타.
_Novelty:_ OP.GG는 전체 서버 기준이지만, 이건 우리 멤버들 사이에서만 통하는 실제 강자를 보여줌.

**[Dashboard #17]: 정보 계층 구조 확정**
_Concept:_ 전체 리그 개요 → 챔피언 드릴다운 → 플레이어 드릴다운 → 매치 상세. 위에서 아래로 좁혀들어가는 탐색 구조.
_Novelty:_ 기간 필터 없음 — 전체 누적 기반. 복잡도 낮추고 일관성 확보.

**[UX #20]: 명예의 전당**
_Concept:_ 자동 감지 조건으로 기록 생성. 최고 Elo, 최다 게임, 연승 기록 등. Dashboard 전체 화면에 고정 섹션으로 노출.
_Novelty:_ 앱을 "기록 보관소"로 인식하게 함 — 단순 통계 뷰어가 아닌 리그의 역사를 담는 공간.

### 테마 3: Electron 대기방 UI/UX

**[Electron #6]: 상대 위협 챔피언 분석**
_Concept:_ 상대 팀 각 플레이어의 고승률 챔피언 TOP 3를 대기방에서 즉시 표시. 해당 유저의 개인 통계 기반.
_Novelty:_ 전체 데이터가 아닌 유저별 개인 승률로 실제 위협도 산정.

**[Electron #7]: 밴 추천 자동화**
_Concept:_ 상대 팀 데이터 기반으로 "지금 밴하면 좋을 챔피언 추천" 리스트를 우선순위와 함께 표시.
_Novelty:_ "상대가 잘 하는 챔피언" × "우리 팀이 상대하기 어려운 챔피언" 교집합으로 밴 가치 계산.

**[Electron #8]: 우리팀 맞춤 픽 추천**
_Concept:_ 현재 팀원들의 개인 고승률 챔피언 풀 기반으로 픽 제안. 팀 내 역할 공백 분석 포함.
_Novelty:_ 개인 승률 + 팀 시너지를 함께 고려한 추천.

**[Electron #9]: 좌우 분할 + 플레이어 카드 레이아웃**
_Concept:_ 화면 좌측 = 우리팀 5명 카드, 우측 = 상대팀 5명 카드. 각 카드에 플레이어명 + 위협 챔피언 + 승률 수치.
_Novelty:_ 롤 클라이언트 대기방과 유사한 구조 차용 — 학습 비용 없음.

**[Electron #10]: 플레이어 카드 구성 확정**
_Concept:_ 닉네임 + Elo + 최근 N게임 기준 고승률 챔피언 + 밴 추천. 10초 안에 스캔 가능한 정보 밀도.
_Novelty:_ 정보 과부하 없이 대기방에서 즉시 판단 가능.

**[Combine #12]: 밴 가치 점수**
_Concept:_ 상대 고승률 챔피언 × 우리팀 해당 챔피언 상대 승률을 결합해 "밴 가치 점수" 하나로 표시.
_Novelty:_ 두 데이터를 머릿속에서 합산하는 인지 부담 제거.

**[Electron #23]: 대기방 자동 트리거**
_Concept:_ 대기방 진입 감지 시 Electron이 자동으로 양팀 분석 화면 표시. 유저가 별도로 앱을 조작할 필요 없음.
_Novelty:_ 데이터가 유저를 찾아옴. 대기방 시간을 최대한 활용 가능.

**[Alien #24]: 밴 추천 명시적 액션 연결**
_Concept:_ 상대 데이터가 있으면 "그래서 뭘 밴해라"는 결론까지 화면에 표시. 데이터 → 인사이트 → 액션 완결 흐름.
_Novelty:_ 데이터를 보여주고 판단을 유저에게 맡기는 기존 구조에서 탈피.

### 테마 4: 경험 확장

**[Alien #26]: 게임 종료 후 자동 요약**
_Concept:_ 게임 종료 후 Electron이 자동으로 "이번 게임 요약"을 표시. Elo 변동, KDA, 팀 기여도 한눈에.
_Novelty:_ 결과 확인을 위해 Dashboard로 갈 필요 없음 — Electron 안에서 경험 완결.
