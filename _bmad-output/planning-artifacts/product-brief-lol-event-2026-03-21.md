# Product Brief: lol-event

<!-- Content will be appended sequentially through collaborative workflow steps -->

## Executive Summary

lol-event는 고정 멤버들의 리그 오브 레전드 내전을 위한 전용 플랫폼이다. 데이터 수집(Electron)과 통계/리포트(Dashboard 웹)를 하나의 생태계로 제공하며, 외부 서비스로는 대체 불가능한 내전 전용 데이터를 기반으로 한다.

이번 프로젝트는 기존 데이터 모델을 유지하면서 Dashboard(웹), Electron 대기방, 모바일 웹 세 영역의 UI/UX를 전면 재설계하여 "데이터는 있는데 불편하다"는 핵심 문제를 해결한다.

---

## Core Vision

### Problem Statement

lol-event의 기존 UI/UX는 세 가지 핵심 문제를 가지고 있다:

1. **탐색성 부족** — 원하는 데이터가 어디 있는지 찾기 어렵다
2. **가독성 부족** — 데이터는 존재하지만 한눈에 파악하기 어렵다
3. **모바일 경험 열악** — 모바일 환경에서의 사용성이 특히 떨어진다

### Problem Impact

내전 전용 플랫폼이므로 유저들은 대안 없이 불편함을 감수하며 사용 중이다. 개선이 이루어질 경우 즉각적이고 직접적인 사용자 경험 향상으로 이어진다.

### Why Existing Solutions Fall Short

OP.GG 등 외부 서비스는 서버 전체 데이터 기반으로, 특정 멤버들 간의 내전 데이터와 맥락을 제공할 수 없다. lol-event는 이 포지션을 독점적으로 차지하고 있다.

### Proposed Solution

기존 데이터 모델과 API는 유지하면서, 세 영역을 백지에서 재설계한다:

- **Dashboard(웹):** 전체→챔피언→플레이어→매치 계층 구조 기반 통계/리포트 허브. OP.GG 스타일 티어표, Elo 리더보드, 명예의 전당 포함.
- **Electron 대기방:** 개인 유저 통계 기반 좌우 분할 카드 레이아웃. 대기방 진입 시 자동 트리거, 밴 가치 점수 + 픽 추천 제공.
- **모바일 웹:** 데스크톱과 명확히 분리된 모바일 최적화 구현.

### Key Differentiators

- **내전 전용 메타 정보:** 서버 전체가 아닌 우리 멤버들 사이의 실제 강자
- **맥락 있는 데이터:** 서로 아는 멤버들의 기록 — 숫자가 감정적 의미를 가짐
- **Electron 자동화:** 대기방 진입 시 분석 자동 제공 — 검색 행동 제거
- **숫자 우선 원칙:** 이모지 없이 정확한 수치 중심의 데이터 친화적 표현

## Target Users

### Primary Users

**유저 타입 1: 일반 참가자 (50-60명)**

내전에 정기적으로 참여하는 멤버들. PC/모바일 구분 없이 상황에 따라 접속.

**주요 니즈:**
- 자신의 통계와 Elo 확인
- 경기 전 상대 파악 (Electron 대기방)
- 챔피언별 승률 등 개인 성적 탐색
- 전체 리그 현황 파악 (리더보드, 티어표)

**현재 불편함:**
- 원하는 데이터를 찾기 어렵고 한눈에 안 들어옴
- 모바일에서 특히 불편함

---

**유저 타입 2: 관리자 (박기준)**

리그를 운영하고 데이터를 관리하는 단일 관리자.

**주요 니즈:**
- 전체 멤버 데이터 기반으로 균형 잡힌 팀 구성
- 리그 데이터 수집/동기화 관리
- 멤버 등록 및 관리

**현재 불편함:**
- 팀 구성 시 여러 멤버의 데이터를 한눈에 비교하기 어렵다
- 관리 작업과 통계 열람이 구분되어 있지 않음

### Secondary Users

N/A — 내전 전용 앱으로 외부 사용자 없음

### User Journey

**일반 참가자의 일반적인 흐름:**

1. **평소:** 모바일/PC로 리더보드, 챔피언 티어표, 개인 통계 열람
2. **내전 당일:** Electron 대기방 자동 실행 → 상대 위협 챔피언 확인 → 밴픽 결정
3. **게임 후:** Elo 변동 및 결과 확인

**관리자의 주요 흐름:**

1. **팀 구성 시:** 전체 멤버 Elo + 챔피언 풀 비교 → 균형 잡힌 팀 편성
2. **리그 운영:** 데이터 수집/동기화, 멤버 관리

## Success Metrics

이 프로젝트는 내부 전용 도구이므로 성공 기준은 사용자 경험 품질과 운영 효율성에 집중한다.

### User Success Metrics

- **가독성:** 멤버들이 원하는 데이터를 추가 탐색 없이 즉시 확인 가능
- **모바일 사용성:** 모바일 환경에서 데스크톱과 동등한 수준의 사용 경험
- **대기방 활용도:** 내전 당일 밴픽 결정 시 Electron 앱을 실제로 참고

### Business Objectives

- Dashboard UI/UX 전면 재설계 완료 (탐색성 + 가독성 개선)
- Electron 대기방 UI 개선 완료 (개인 통계 기반 밴픽 추천)
- 모바일 웹 최적화 완료 (데스크톱과 분리된 모바일 전용 구현)

### Key Performance Indicators

- 관리자가 팀 구성 시 한 화면에서 전체 멤버 데이터 비교 가능
- 대기방 진입 시 Electron 자동 트리거로 별도 조작 불필요
- 모바일에서 주요 화면(리더보드, 개인 통계, 챔피언 티어표) 불편함 없이 사용 가능

## MVP Scope

### Core Features

**1. Dashboard UI/UX 전면 재설계**
- 전체→챔피언→플레이어→매치 계층 구조 기반 탐색 구조
- OP.GG 스타일 내전 챔피언 티어표 (우리 리그 데이터 기반)
- Elo 리더보드 (홈에서 즉시 접근)
- 챔피언 밴픽 트렌드 표시
- 숫자 우선 표현 원칙 전면 적용

**2. Electron 대기방 UI 개선**
- 대기방 진입 시 자동 트리거
- 좌우 분할 플레이어 카드 레이아웃 (우리팀 ↔ 상대팀)
- 카드 구성: 닉네임 + Elo + 개인 고승률 챔피언 + 밴 추천

**3. 모바일 웹 최적화**
- 데스크톱과 명확히 분리된 모바일 전용 구현
- 주요 화면(리더보드, 개인 통계, 챔피언 티어표) 모바일 최적화

### Out of Scope for MVP

- 명예의 전당 (역대 기록 자동 감지)
- 게임 종료 후 자동 요약
- 밴 가치 점수 고도화

### MVP Success Criteria

- 멤버들이 원하는 데이터를 추가 탐색 없이 즉시 확인 가능
- 모바일에서 주요 화면 불편함 없이 사용 가능
- 대기방에서 밴픽 결정 시 Electron 앱을 실제로 참고

### Future Vision

- 명예의 전당: 역대 기록 자동 감지 및 Dashboard 고정 노출
- 게임 종료 후 Electron 자동 요약 (Elo 변동 + KDA)
- 밴 가치 점수 고도화 (상대 승률 × 우리팀 상대 승률 결합)
